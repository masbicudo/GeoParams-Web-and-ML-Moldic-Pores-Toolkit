# User-Parameter Porosity

This subproject applies previously collected user segmentation parameters to
new petrographic images. It does not collect new user data and does not train
machine-learning models.

## Setup

Run commands from this directory:

```bash
./install.sh
pdm run python imports.py
```

`imports.py` copies the anonymized parameter table generated by
`geo_params_web` into `data/c_min_k_max_params.csv`.

## Input Images

Place images to analyze in a directory such as:

```text
data/reviewer_new_images/images/
```

If the images include borders or non-thin-section areas, provide a crop
metadata file:

```json
{
  "image-name.jpg": {
    "x": 100,
    "y": 200,
    "width": 1200,
    "height": 900
  }
}
```

Coordinates are in pixels in the original image.

## Example

```bash
pdm run python measure_porosity_from_params.py \
  "data/reviewer_new_images/images/*" \
  --crop-metadata data/reviewer_new_images/crop-metadata.json \
  --output-dir data/output/reviewer_new_images_original_method
```

Outputs:

- `porosity_summary.csv` - one row per image
- `porosity_by_parameter.csv` - one row per image and user parameter pair
- `mean_mask_*.png` - mean user-parameter binarization masks
