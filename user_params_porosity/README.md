# User-Parameter Porosity

This subproject applies previously collected user segmentation parameters to
new petrographic images. It does not collect new user data and does not train
machine-learning models.

## Setup

Run commands from this directory:

```bash
./install.sh
pdm run python quick_test.py
```

`quick_test.py` copies the anonymized parameter table generated by
`geo_params_web`, analyzes one public thin-section image, and prints a short
summary.

## Input Images

The default scripts expect the public dataset at the repository root:

```text
../datasets/
  article_thin_sections/
  generalization_test_thin_sections/
  pore_type_training/
```

The generalization-test crop metadata lives in this subproject:

```text
data/cut-metadata.json
```

The article thin-section crop metadata is reused from the web application
image-preparation cache.

## Default Analyses

Run the article thin sections:

```bash
pdm run python run_article_thin_sections.py
```

Run the generalization-test thin sections:

```bash
pdm run python run_generalization_test.py
```

Outputs are written under:

```text
data/output/
```

## Crop Metadata Format

Crop metadata maps image filenames to rectangles:

```json
{
  "image-name.jpg": {
    "x": 100,
    "y": 200,
    "width": 1200,
    "height": 900
  }
}
```

Coordinates are in pixels in the original image.

## Advanced Usage

```bash
pdm run python measure_porosity_from_params.py \
  "../datasets/generalization_test_thin_sections/*" \
  --crop-metadata data/cut-metadata.json \
  --output-dir data/output/generalization_test_thin_sections
```

Outputs:

- `porosity_summary.csv` - one row per image
- `porosity_by_parameter.csv` - one row per image and user parameter pair
- `mean_mask_*.png` - mean user-parameter binarization masks
